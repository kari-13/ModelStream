# Security Policy

## Supported Versions

We actively support security updates for the following versions:

| Version | Supported |
|--------|----------|
| latest | ✅ Yes |
| older versions | ❌ No |

If you are using an older version, please upgrade to the latest release.

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project (terminal Markdown renderer built around :contentReference[oaicite:0]{index=0}), please report it responsibly.

### Do NOT:
- Open a public issue
- Discuss it in public forums
- Share it on social media

---

### DO:
Open a **private security advisory** via GitHub Security Advisories (if enabled for this repository).

---

## What to Include

Please include:

- Description of the vulnerability
- Steps to reproduce
- Affected version(s)
- Potential impact
- Proof of concept (if available)

---

## What Happens Next

After you report a vulnerability:

1. We will acknowledge the report
2. We will investigate and validate the issue
3. We will develop a fix
4. We will coordinate responsible disclosure before public release

---

## Scope

This security policy applies to:

- Core Markdown rendering engine
- CLI tools
- Streaming handling for Ollama-based output
- Repository-provided utilities

Out of scope:

- Third-party dependencies
- External services (including Ollama)
- Forks or modified versions of the project

---

## Responsible Disclosure

We request that vulnerabilities are not publicly disclosed until a fix has been released and users have had time to update.
